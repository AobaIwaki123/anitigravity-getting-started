# Google Antigravity 調査報告書

## 1. 概要
Google Antigravityは、Googleが2025年11月に発表した新しいAI搭載統合開発環境（IDE）です。Gemini 3 Proモデルを中核に据え、「Agent-first（エージェントファースト）」という新しいパラダイムを採用しています。従来の「AIがコード入力を支援する」形から、「AIエージェントが自律的に計画・実行・検証を行う」形へと進化させています。

**情報の信頼度について:**
- **高 (公式情報):** Google公式ブログ, Antigravity公式サイト
- **中 (著名技術メディア):** Wikipedia, Codecademy, Medium等の技術解説記事
- **低:** (今回の調査では低信頼度の雑多な記事は主要な情報源として使用していません)

## 2. 従来のAI IDEとの共通点と相違点

### 共通点
- **VS Codeベース:** Visual Studio Codeのフォークとして構築されており、UIや操作感は従来のVS Codeユーザーにとって馴染み深いものです。
- **AIによるコード支援:** コード補完、チャットによる質問応答、リファクタリング支援など、CursorやGitHub Copilotと同様の基本機能も備えています。

### 相違点 (Antigravityの独自性)
- **エージェントファースト (Agent-First):**
  - **従来:** ユーザーが主体で、AIは「副操縦士 (Copilot)」として都度指示を受ける。
  - **Antigravity:** AIが「自律的なエージェント」として振る舞う。ユーザーは大まかなゴール（例：「認証機能を追加して」）を指示し、エージェントがタスク分解、計画立案、複数ファイルへの変更、テスト実行までを自律的に行う。
- **Artifacts (成果物) による透明性:**
  - 単にコードを変えるだけでなく、「タスクリスト」「実装計画書」「ブラウザ操作の録画」などの成果物 (Artifacts) を生成し、ユーザーがプロセスの正当性を検証できるようにしている。
- **マルチエージェント協調:**
  - Planner（計画）、Coder（実装）、Reviewer（レビュー）、Execution（実行）といった役割分担された複数のエージェントが協調してタスクを遂行する。
- **統合された実行環境:**
  - エディタだけでなく、ターミナルやブラウザもAIが直接操作できる。AI自身がブラウザを開いて動作確認を行い、その様子を録画してユーザーに提示することが可能。

## 3. Antigravityでできること (機能列挙)

- **自律的なタスク計画と実行:** 自然言語での指示から、必要な手順をリスト化し、順次実行する。
- **複数ファイルにまたがるリファクタリング:** プロジェクト全体を理解し（Gemini 3の長大なコンテキストウィンドウを活用）、依存関係を考慮した大規模な変更を行う。
- **自律的なデバッグと修正:** エラーが発生した場合、AIが自らログを読み、原因を特定し、修正コードを適用して再テストを行う。
- **ブラウザ操作によるE2Eテスト:** AIが内蔵ブラウザを操作してWebアプリの動作を確認し、クリックや入力のテストを行う。
- **成果物 (Artifacts) の生成:**
  - 実装計画書 (Implementation Plan)
  - タスクリスト (Task List)
  - 変更内容のDiff
  - テスト実行のスクリーンショットや動画
- **Manager Viewによる管理:** 複数のエージェントが並行して作業している状況を俯瞰し、管理する専用のビューを提供する。

## 4. Antigravityを使用するリスク

- **過度な抽象化によるブラックボックス化:** AIが自律的に動きすぎるため、詳細な実装内容をユーザーが把握しきれなくなる可能性がある（Artifactsで緩和されているが、チェック漏れのリスクは残る）。
- **コストと依存:** プレビュー期間中は無料だが、Gemini 3 Proのような高性能モデルを大量に使用するため、将来的なコスト構造が不透明。また、Googleのエコシステムへの依存度が高まる。
- **AIの幻覚 (Hallucination) の影響範囲:** AIが誤った計画を立てた場合、自律的に実行してしまうため、従来よりも広範囲に誤ったコードが拡散するリスクがある（Reviewerエージェントがいるとはいえ完全ではない）。
- **プレビュー版としての不安定さ:** リリース直後であるため、動作の不安定さや仕様変更の可能性がある。

## 5. Antigravityを用いて可能な技術的に興味深い挑戦

- **仕様書からの完全自動実装:**
  - 詳細な要件定義書やデータシート（PDFなど）を読み込ませ、ドライバコードやAPIクライアントをゼロから実装・テストまで完結させる。
- **自己修復型システムの構築:**
  - CI/CDパイプラインに組み込み、テストが落ちたらAntigravityが自律的にコードを修正してコミットし直す「人間が寝ている間にバグが直る」ワークフロー。
- **レガシーコードの自律的モダナイゼーション:**
  - 「この古いJavaのコードベースを最新のGo言語に書き換えて」といった指示で、言語間の翻訳だけでなく、ライブラリ選定やアーキテクチャの変更まで含めた移行プロジェクトを遂行させる。
- **動的なUI/UXの探索的生成:**
  - 「もっと使いやすいデザインにして」という抽象的な指示に対し、複数のデザイン案をコードとして実装し、それぞれのプレビューを生成してユーザーに選ばせる（A/Bテストの準備まで自動化）。
